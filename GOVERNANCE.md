# Governance

LeftLevel Helix is currently maintained under the LeftLevel project.

## Project steward

The project steward is the LeftLevel project owner. The steward is responsible for:

- product direction;
- release decisions;
- license decisions;
- maintainer access;
- security disclosure handling;
- brand and trademark decisions.

## Maintainers

Maintainers may be added as the project grows. Maintainers should be trusted contributors who understand the security model and documentation standards.

Maintainer responsibilities:

- review pull requests;
- keep tests passing;
- avoid unsupported security claims;
- protect user privacy;
- enforce contribution standards;
- escalate security-sensitive changes.

## Decision process

For now, decisions are made by maintainer consensus when possible, with the project steward making the final decision when needed.

## Security-sensitive changes

The following require explicit maintainer approval:

- cryptographic changes;
- protocol changes;
- relay metadata changes;
- identity and verification changes;
- storage of secrets;
- dependency additions;
- release signing;
- license changes.

## Commercial direction

The open-source project can coexist with commercial offerings such as hosted relays, paid support, audits, managed deployments, training, and enterprise integrations. Commercial offerings should not misrepresent the maturity of the open-source prototype.

## Brand use

The LeftLevel name, logo, and related branding should not be used to imply endorsement or official status without permission from the project steward.
